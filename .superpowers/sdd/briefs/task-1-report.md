## Issues encountered and resolutions

1. **`npx tsc --noEmit` failed on CSS side-effect import.**
   - Error: `src/main.tsx(4,8): error TS2882: Cannot find module or type declarations for side-effect import of './styles/globals.css'.`
   - Cause: The brief's `tsconfig.json` does not include Vite's client types, so TS has no declaration for `*.css` imports.
   - Resolution: Added `src/vite-env.d.ts` with `/// <reference types="vite/client" />`. This is the standard Vite convention and provides type declarations for CSS/asset imports. This is a supporting file not listed in the brief but necessary for the stated success criterion (`npx tsc --noEmit` passes). After adding it, tsc passes with exit 0.

2. **Installed dependency versions newer than brief examples.**
   - npm resolved to current latest: TypeScript 6.0, Vite 8.1, Tailwind 4.3, React 19.2.
   - All config files were written verbatim from the brief. `cargo check` and `npx tsc --noEmit` both pass with these versions, so no action needed. (Note: Tailwind v4 changes the `@tailwind` directive approach, but since the task's success criteria are limited to cargo check + tsc + file existence, this was not addressed. The CSS will need migration to `@import "tailwindcss"` in a future task if Tailwind v4 is kept.)
